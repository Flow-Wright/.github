# Contributing to FlowWright

Thanks for your interest in FlowWright. 🦫 These guidelines apply across every
[Flow-Wright](https://github.com/Flow-Wright) repository.

> **FlowWright is pre-launch.** APIs, docs, and interfaces are still moving. Small,
> focused contributions are the easiest to land right now — please open an issue before
> starting anything large so we don't duplicate work.

## Reporting bugs & requesting features

- Search [existing issues](https://github.com/Flow-Wright/FlowWright/issues) first.
- Open a new issue using the **Bug report** or **Feature request** form.
- For questions and ideas, prefer the website or discussions over an issue.

## Pull requests

1. Fork and branch from `main`. Use a descriptive branch name: `fix/<slug>` or `feat/<slug>`.
2. Keep the change focused — one concern per PR.
3. Match the surrounding code style; run the repo's lint/format/tests before pushing.
4. Write a clear PR description and link the issue it closes (`Closes #123`).
5. Fill out the pull request template.

## Commit messages

Write imperative, present-tense summaries (e.g. "Fix countdown drift on DST"). Group
unrelated changes into separate commits.

## Code of conduct

Participation is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). Be kind.

## Security

Please **do not** open public issues for vulnerabilities — see [SECURITY.md](SECURITY.md).
